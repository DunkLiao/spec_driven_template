# 用戶需求轉規格 範例

此範例示範如何將產品需求 (PRD) 轉換為清晰的技術規格，包含範圍、介面、與驗收標準。

## 目的

將語意模糊的需求明確化，提供給開發與測試團隊可執行的規格。

## 範例內容

- 功能：使用者可以新增商品至購物車
- 介面：POST /cart/add
- 驗收標準：當商品庫存充足且資料格式正確，回傳 200 與購物車摘要

```yaml
endpoint: POST /cart/add
request:
  body:
    productId: string
    quantity: integer
responses:
  200:
    description: 成功
```
