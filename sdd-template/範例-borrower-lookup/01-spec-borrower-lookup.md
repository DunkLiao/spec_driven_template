# 規格：授信戶風險分級查詢

> 階段一：講清楚「做什麼」，不講「怎麼做」。

- 狀態：已定案
- 最後更新：2026-06-26

---

## 1. 問題敘述 (Why)

同仁目前要查單一授信戶的風險分級，得開啟大型系統、逐層點選，平均要一兩分鐘。
臨時口頭詢問（例如主管走過來問「這戶現在幾級？」）時，操作太慢。
需要一個輸入統編就立刻回傳分級的輕量查詢頁。

## 2. 目標與非目標 (Scope)

**目標（這次要做）**
- 輸入統編，回傳該戶風險分級（1～5 級）與最後更新日。
- 查無資料時顯示明確提示。
- 統編格式錯誤時即時提醒，不送出查詢。

**非目標（這次不做）**
- 不做批次查詢 / 匯入。
- 不做歷史分級變動軌跡。
- 不做登入與權限管理（限內網、後續再議）。

## 3. 使用者故事 (User Stories)

- US-1：身為風管部同仁，我想要輸入統編就查到風險分級，以便臨時詢問時即刻回答。
- US-2：身為風管部同仁，我想要在查無資料或輸入錯誤時得到清楚提示，以便知道是打錯還是真的沒這戶。

## 4. 驗收標準 (Acceptance Criteria)

### US-1
- **AC-1.1**：假設統編 12345678 在資料中分級為「第 3 級」，當使用者輸入 12345678 並查詢，則畫面顯示「第 3 級」與最後更新日。
- **AC-1.2**：假設查詢成功，當結果顯示時，則同時顯示該統編，避免看錯對象。

### US-2
- **AC-2.1**：假設輸入非 8 碼數字，當使用者輸入時，則即時提示「統編須為 8 碼數字」且查詢鈕停用。
- **AC-2.2**：假設統編格式正確但查無資料，當查詢，則顯示「查無此授信戶資料」。

### 邊界與例外
- 輸入為空 → 查詢鈕停用。
- 後端錯誤 → 顯示「系統忙碌，請稍後再試」，不顯示技術錯誤訊息。
